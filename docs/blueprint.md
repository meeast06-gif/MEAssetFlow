# **App Name**: AssetWatch

## Core Features:

- Asset Input Form: Allow users to input new asset details such as name, type, current value, acquisition date, and notes.
- Asset List View: Display a comprehensive list of all added assets with key information in a clean, tabular format.
- Asset Detail & Edit: View individual asset details and allow users to update or delete existing asset information.
- Asset Summary Dashboard: Provide a simple overview of the total asset value and a basic breakdown by asset type.
- Firestore Data Storage: Securely store and retrieve all asset data in a NoSQL database for persistence across sessions.
- AI Asset Insights Tool: A generative AI tool that analyzes the entered assets and provides a high-level summary of the user's current portfolio distribution and key characteristics.

## Style Guidelines:

- Primary color: A sophisticated, slightly muted blue (#2E6ECC) to evoke trust and stability for financial tracking. (HSL: 220, 60%, 45%)
- Background color: A very light, almost off-white hue of the primary color (#ECF0F2) for a clean, legible base. (HSL: 220, 15%, 95%)
- Accent color: A vibrant teal-like blue (#55C8E6) to draw attention to interactive elements and highlight data points, analogous to the primary color. (HSL: 190, 70%, 60%)
- Headline and body font: 'Inter', a modern, neutral sans-serif that ensures high readability and a clean, objective aesthetic for financial data.
- Utilize simple, outline-style icons to represent different asset types (e.g., house, bank, stock chart) for quick visual identification without clutter.
- Employ a clean, structured layout with ample whitespace, utilizing a responsive grid system to maintain clarity and usability across various screen sizes.
- Implement subtle fade-in and slide-in animations for new data loads or transitions between views, providing a smooth and professional user experience without being distracting.
